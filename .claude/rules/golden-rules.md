# Golden Rules — GiuOS

Estas regras não são sugestões. São o comportamento base do agente em qualquer projeto GiuOS.

1. **Nunca codificar sem SPEC aprovada por Giuliana.**
2. **Nunca avançar de fase sem aprovação explícita.**
3. **Ler antes de agir** — antes de qualquer ação, entender o estado atual do projeto.
4. **Verbalizar o raciocínio** — antes de propor qualquer coisa, explicar o porquê usando o vocabulário metodológico correto.
5. **Separar fato, inferência, hipótese, e aberto** — nunca inventar dados de mercado, personas, ou validações.
6. **PRD antes de Shaping. SPEC antes de Building.** Sem exceções.
7. **Session log para decisões materiais** — qualquer decisão que afeta direção do produto, stack, ou escopo deve ser registrada.
8. **Frontmatter YAML em todos os documentos** — campos estruturados para o agente, corpo narrativo para humanos.
9. **Figma primeiro para UI** — verificar referência no Figma via MCP antes de gerar qualquer componente visual.
10. **Código modular** — arquivos abaixo de 200 linhas onde prático, organizado por feature, sem dependências cruzadas ocultas.
11. **Rabbit holes são sinalizados, não implementados** — se algo não estava na SPEC, sinalizar antes de agir.
12. **Giuliana é a dona do produto** — o agente propõe, ela decide.
